Overview

What It Does

AMVIS is a real-time conversational AI platform built as a 7-service microservice system that runs a full ASR → LLM → TTS voice conversation loop. Each turn of the conversation flows through automatic speech recognition, language model inference, and text-to-speech synthesis — all coordinated through asynchronous message streams with per-session state.

Audio chunks and LLM token streams are passed between services using Redis Streams, enabling low-latency, backpressure-aware communication without tight coupling between services. This architecture allows each service to scale independently and fail in isolation.

Session transcripts, metadata, and telemetry are persisted in MongoDB, providing a complete audit trail for every conversation. The React frontend handles live audio capture via the Web Audio API and renders a real-time transcript as tokens stream in. After each session, a post-session analytics view surfaces conversation quality metrics and turn-level timing data.

All 7 services are containerized with Docker Compose for local development and deployed on self-hosted infrastructure via Coolify, with each service having its own image, environment, and health-check configuration.

Features

Key Components

Real-Time ASR
faster-whisper handles speech recognition with chunked audio streaming, producing low-latency transcripts as the user speaks rather than waiting for complete utterances.
Stateful LLM Orchestration
Per-session conversation state managed through Ollama with streaming token output. The orchestrator service maintains context windows and routes turn-level messages between pipeline stages.
TTS with OmniVoice
OmniVoice converts LLM output to natural-sounding speech, streaming audio back to the frontend as chunks are generated for minimal perceived latency.
Redis Streams Messaging
Audio chunks and token streams flow between the 7 services via Redis Streams, providing durable, ordered message delivery with consumer group semantics and built-in backpressure.
MongoDB Persistence
Session transcripts, turn-level timestamps, service latency telemetry, and conversation metadata are persisted in MongoDB for post-session analysis and system observability.
React Frontend with Analytics
Web-based interface built with React handles live audio capture via Web Audio API, displays real-time transcript as tokens arrive, and renders a post-session analytics dashboard with conversation quality metrics.
