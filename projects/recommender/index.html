Overview

What It Does

The University Recommender is a two-stage retrieval and re-ranking system that helps prospective students find relevant universities based on their profile, preferences, and academic background. It combines classical and neural retrieval methods with a transparent multi-signal re-ranking layer.

Stage 1 — Candidate Retrieval: Two retrieval backends are used in parallel. A TF-IDF retriever handles keyword-sensitive queries (specific programs, locations, rankings), while a sentence-transformer embedding retriever captures semantic similarity for more abstract or profile-based queries. Candidates from both backends are pooled and deduplicated.

Stage 2 — Re-Ranking: Retrieved candidates are scored using a multi-signal re-ranking layer that combines retrieval scores with structured signals — academic fit, program match, location preference, tuition bands, and ranking tiers. Weights are tunable per deployment context, making the re-ranking fully transparent and interpretable.

The system was benchmarked using Recall@K, MRR (Mean Reciprocal Rank), and nDCG@K across both retrieval backends and combined configurations to select the deployed setup. Results are served via a FastAPI endpoint that returns ranked lists with score breakdowns for each candidate.

Retrieval Benchmarks

Retrieval backends were benchmarked across three complementary IR metrics to select the deployed configuration.

Recall@K
Coverage — fraction of relevant items in top-K results
MRR
Mean Reciprocal Rank — first relevant result position quality
nDCG@K
Normalized Discounted Cumulative Gain — ranked list quality
Features

Key Components

TF-IDF Retrieval
Sparse keyword retrieval using TF-IDF (Scikit-learn) excels on exact-match and keyword-sensitive queries — program names, locations, rankings — where lexical overlap is the right signal.
Sentence-Transformer Embeddings
Dense semantic retrieval using sentence-transformer models captures latent similarity between user profiles and university descriptions — handling paraphrase, intent, and concept-level matching beyond keyword overlap.
Multi-Signal Re-Ranking
A transparent scoring layer combines retrieval relevance scores with structured signals: academic profile fit, program offering match, location preference, tuition range, and global ranking tier — all with tunable weights.
IR Metric Benchmarking
Systematic evaluation of TF-IDF-only, embedding-only, and combined retrieval configurations using Recall@K, MRR, and nDCG@K on a held-out query set to select the best deployed configuration.
Interpretable Results
API responses include per-candidate score breakdowns showing each signal's contribution to the final ranking, making recommendations explainable to end users rather than opaque black-box outputs.
FastAPI Endpoint
Clean REST API that accepts user profile payloads, runs the two-stage retrieval and re-ranking pipeline, and returns a ranked list of university recommendations with score breakdowns and metadata.
